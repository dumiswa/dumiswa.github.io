About

A three-part project that I developed for an university assignment, exploring algorithmic dungeon generation and traversal. It begins with binary space partitioning to generate rooms, followed by graph-based representation, and ends with interactive pathfinding via recursive and breadth-first algorithms.

Project Info

  • Role: Engineer
  • Team Size: 1
  • Duration: 4 weeks
  • Framework: GXPEngine (C#)

Binary Space Partitioning

Rooms are generated by recursively splitting a 2D space, alternating between vertical and horizontal directions. Each room is defined by a rectangle and added to a list if too small to split further. After generation, rooms with the largest and smallest surface areas are removed.

Graph Construction

I converted each room and door into a node in a high-level node graph. The center of every room and the position of every door are used to define node positions. Connections are established between room nodes and their corresponding door nodes if the door lies within the room area.

Tile-Based Visualization

A tile grid is generated where all tiles are initially set to walls. I made the ground tiles painted inside room boundaries with walls around them. Door positions are set as ground tiles, integrating them visually with surrounding rooms.

Pathfinding Algorithms

The system includes a recursive pathfinder and a breadth-first pathfinder:

  • Recursive Pathfinding: Recursively explores all possible routes between nodes, backtracking to find the shortest route by node count.
  • Breadth-First Search: Iteratively uses a queue and parent mapping to ensure the shortest path is found using the fewest number of nodes.

Paths are visually executed step-by-step in response to player input.

What I Learned

This project helped me understand procedural content generation, graph construction, and unweighted pathfinding algorithms works. I practiced recursion, queue-based logic, and spatial reasoning across interconnected systems.
