IceAge

About

I made this small OpenGL demo to practice fragment and vertex shaders from scratch. I wanted to create a visual effect without any textures, just math, so I ended up building a full-screen procedural checkerboard pattern using GLSL. It is a clean little project where the entire output is generated directly by the GPU.

Project Info

  • Role: Engineer
  • Team Size: 1
  • Duration: 3 weeks
  • Engine: C++, OpenGL, GLSL

Rendering Pipeline

I kept the setup simple, just a single fullscreen triangle rendered with no transformations. I used OpenGL with a C++ wrapper to initialize everything, then passed it over to the shader. Most of the heavy lifting was done in the fragment stage. I did not want to deal with VAOs or complex meshes, so I kept the focus on writing clean, minimal GLSL.

Vertex Shader

In the vertex shader, I just passed through static positions for a big fullscreen triangle. There was no need for any transformations or lighting — the only goal here was to get something on screen so I could play with the fragment stage. It is as bare-bones as it gets, but it does exactly what I needed it to do.

Fragment Shader

This is where everything happens. I wrote a checkerboard pattern directly in the fragment shader using `mod` and `floor` operations on the UV coordinates. The cool part is that it's resolution-independent and does not use any textures at all. I also made sure it scales nicely with window size and aspect ratio, so it feels consistent no matter how you resize the window.

Checkerboard Logic

The pattern logic is pretty straightforward but surprisingly fun to implement. I take the screen coordinates, scale them into a grid, and alternate black and white squares using a simple XOR trick on the grid indices. It is satisfying how a couple of lines of math can produce a crisp and clean effect that fills the screen. This type of approach is perfect for procedural graphics or tools that need lightweight patterns on the fly.

What I Learned

I learned a lot about how shaders are structured in OpenGL — especially how fragment shaders can do way more than I initially thought. I also got more comfortable writing raw GLSL, and honestly, just seeing an image come to life from pure math felt super rewarding. I definitely want to explore more visual experiments like this in the future, maybe even play with noise functions or dynamic input.
